# Progress Report 2026-04-03

## Summary

Major APLArray refactoring day. Made APLArray abstract, created NumpyArray subclass, migrated all monadic and dyadic primitives to methods, eliminated functions.py, refactored operator_binding to glyph-based lookup, and replaced ufunc lookups with numpy operators for ulab compatibility. Version 0.5.51 → 0.6.17.

## What was done

### APLArray factory methods

- Added `APLArray.array(shape, data)` and `APLArray.scalar(value)` class methods
- Converted all 170 `APLArray(` constructor calls in tests to use factory methods
- Converted all ~120 constructor calls in production code to use factory methods
- Bumped to v0.6.0 as safe revert point before abstraction

### APLArray made abstract with NumpyArray subclass

- `APLArray` is now an ABC in `backend.py` with `@abstractmethod` signatures for all primitives
- `NumpyArray` subclass implements all methods
- Factory methods create `NumpyArray` instances
- `arraymodel.py` is a thin re-export for backward compatibility

### All monadic primitives migrated (20 methods)

- Arithmetic: conjugate, signum, negate, reciprocal, ceiling, floor
- Extended: exponential, natural_log, absolute_value, logical_not, pi_times, factorial
- Structural: shape_of, ravel, reverse, reverse_first, transpose, matrix_inverse
- Env-dependent: iota, tally, grade_up, grade_down, roll, format

`monadic_functions.py` is now a thin dispatch layer — all `_SIMPLE` entries are lambda→method calls, `_ENV_DEPENDENT` entries pass io/ct to methods. No imports from functions.py or structural.py.

### All dyadic primitives migrated (31 methods)

- Arithmetic: add, subtract, multiply, divide, maximum, minimum, power, logarithm, residue
- Extended: circular, binomial
- Comparisons: less_than, less_equal, equal, greater_equal, greater_than, not_equal (with ct parameter)
- Logical: logical_and, logical_or
- Match: match, not_match
- Deal: deal
- Structural: reshape, catenate, take, drop, rotate, rotate_first, encode, decode, replicate, replicate_first, expand, matrix_divide, index_of, membership, from_array
- Format: dyadic_format

### functions.py deleted

All standalone function definitions removed. The file no longer exists. Primitive implementations live on NumpyArray in backend.py, with structural operations delegating to structural.py.

### operator_binding refactored

- Identity elements and dispatch now keyed by glyph string, not function object identity
- Removed ufunc fast path from reduce (methods handle numpy directly)
- No imports from functions.py

### Numpy operators replace ufunc lookups

- Monadic: `-self.data` instead of `np.negative`, `abs(self.data)` instead of `np.absolute`, etc.
- Dyadic: `self.data + other.data` instead of `np.add` ufunc lookup
- `_numeric_dyadic_op` helper handles overflow protection and shape mismatch errors
- Comparisons use vectorised tolerant equality with `np.maximum`/`abs`
- `_dyadic` simplified: removed ufunc path, now just element-wise Python fallback
- `_OVERFLOW_UFUNCS` removed (dead code)
- Fixes 4 ulab failures: `np.negative`, `np.sign`, `np.absolute`, `np.logical_not` don't exist in ulab

## Architecture after refactoring

```
APLArray (ABC in backend.py)
  ├── abstract method signatures for all 51 primitives
  ├── factory methods: array(), scalar()
  └── NumpyArray (concrete subclass)
       ├── monadic methods (20)
       ├── dyadic methods (31)
       ├── _numeric_dyadic_op (numpy fast path)
       ├── _dyadic (Python fallback for circular, binomial, non-numeric)
       └── _compare (vectorised tolerant equality)

monadic_functions.py — thin dispatch (lambda→method)
dyadic_functions.py — thin dispatch (lambda→method)
operator_binding.py — glyph-based identity/dispatch
structural.py — complex structural implementations (called by NumpyArray methods)
```
