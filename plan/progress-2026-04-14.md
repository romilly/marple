# Progress Report — 2026-04-14

## Summary

Large structural refactor of the type hierarchy plus two Dyalog
divergences fixed along the way. 19 commits, v0.8.20 → v0.8.39,
+875 / −597 lines across 17 files. 1343 fast + 8 slow tests green
throughout; pyright clean at every commit.

## Apply_derived cleanup (v0.8.21)

Collapsed `Executor.apply_derived` from a tangled 10-line
isinstance chain to two lines. The outer
`isinstance(function, Evaluatable)` was a tautology; the `DfnBinding`
lambda branch was dead (always rejected by
`DerivedFunctionBinding._resolve_function`). `DerivedFunc.function`
field type tightened from `object` to `Evaluatable`.

## First-class Function and Operator types (v0.8.22 → v0.8.30)

APL has three first-class categories — Array, Function, Operator —
but marple only represented Array cleanly. Function-ness was
scattered across ~10 classes held together by `UnappliedFunction`
and `Applicable`, and Operator had no runtime type at all.

Staged in three groups (A, B, C) over nine commits:

- **A1 / A2**: Introduced `Function(APLValue)` and `Operator(APLValue)`
  abstracts; bridged existing types so they inherit the new abstracts.
- **B1**: `FunctionRef` → `PrimitiveFunction`. Convenience methods
  (apply_to_monadic, call_monadic, as_power_strategy) moved from
  `UnappliedFunction` onto `Function`.
- **B2**: Primitive operators (`Adverb`, `Conjunction`) gained
  `derive_monadic` / `derive_dyadic` methods. Parser's `_bound_to_derived`
  delegates to the operator polymorphically.
- **B3**: Split `DopBinding(Operator)` out of `DfnBinding(Function)`.
  `Dfn.is_operator()` picks the right binding class at creation time.
  User-defined operators are no longer indistinguishable from
  user-defined functions.
- **B4**: Eliminated `Applicable` ABC entirely. Became a type alias:
  `Applicable = Function | Evaluatable`. Python 3.10+ union-isinstance
  preserves runtime checks.
- **C1**: Collapsed `_BOUND_MONADIC_DISPATCH` / `_BOUND_DYADIC_DISPATCH`
  tables. Six one-line helper methods deleted.
- **C2**: `_make_monadic` / `_make_dyadic` 10-class isinstance tuple
  collapsed to `isinstance(verb_node, (Evaluatable, UnappliedFunction))`.
- **C3**: Call sites naming concrete types (`DfnBinding`, `IBeamDerived`)
  switched to the abstract `Function` / `Operator`.

## Per-glyph operator subclasses (v0.8.31)

Symbol-string dispatch inside `Adverb.derive_monadic` /
`Conjunction.derive_dyadic` eliminated via per-glyph subclasses:
`CommuteAdverb`, `ReduceAdverb`, `ScanAdverb`, `RankConjunction`,
`PowerConjunction`, `BesideConjunction`. Factory functions
(`make_adverb`, `make_conjunction`) pick the right subclass.

## Inner/outer product as stored functions (v0.8.32)

**Bug fix**: `f ← +.×` and `f ← ∘.×` previously raised DomainError
at assignment time (Dyalog accepts). New `InnerDerived(Function)` /
`OuterDerived(Function)` with `apply_dyadic`. `InnerProductConjunction`
and `OuterProductAdverb` produce them via `derive_*`. The old
`InnerProduct` / `OuterProduct` AST call-nodes are deleted — direct
call and assignment-then-call now share one code path.

## I-beam reworked per Dyalog (v0.8.33)

Previously `⌶'module.function.path'` (string Python import). Now
`(A⌶)` where A is an integer selecting a built-in service, matching
Dyalog's `R←{X}(A⌶)Y` spec. New `IBeamAdverb(Adverb)` +
`IBeamFunction(Function)` + `src/marple/ibeam_registry.py` module.
Codes 100/101/102 wrap `str_impl.upper/lower/trim` for test coverage.
`IBeamDerived`, `Executor.call_ibeam`, and the parser's string-path
special case are deleted.

## Dead Adverb/Conjunction state removed (v0.8.34)

Base `Adverb` and `Conjunction` no longer carry a symbol field —
dead payload on every single-glyph subclass. Only `ReduceAdverb`
and `ScanAdverb` keep it (load-bearing for `/` vs `⌿`, `\` vs `⍀`).
Factory functions use per-entry lambdas so each constructs with the
right arguments.

## Reduce/scan with axis specifier (v0.8.35)

**Feature + bug fix**. `f/[k] Y`, `f\[k] Y`, `f⌿[k] Y`, `f⍀[k] Y`
now parse and evaluate. `ReduceAdverb` and `ScanAdverb` gained an
optional axis `Evaluatable`; parser peeks for `[...]` after the
operator. Backend `_reduce` / `_scan` unified into single
axis-parameterised functions using `np.moveaxis`; `_reduce_first`
and `_scan_first` deleted.

Pre-existing bug fixed as a side effect: first-axis reduce/scan
(`⌿`, `⍀`) was left-to-right instead of APL's right-to-left.
Exposed by `-⌿ 3 2⍴⍳6`: marple `-7 -8`, Dyalog `3 4`. New generic
backend is right-to-left along any axis. 14 new tests in
`test_axis.py`, including the `-⌿` / `-⍀` regression.

## Evaluatable → Executable rename (v0.8.36)

Pure rename. The defining abstract method is `execute()`, so
`Executable` reads more naturally. 147 references across 5 source
files; no test references.

## DomainError import hoist (v0.8.37)

Eight inline `from marple.errors import DomainError` imports in
`APLValue`, `Operator`, and `PowerByCount` replaced with a single
module-level import. `errors.py` has no marple imports — no
circular-import risk.

## APLValue slimmed (v0.8.38)

`APLValue` carried seven default-raise methods (apply_to_*, call_*,
apply_*_dop, as_power_strategy) as a safety net for callers that
handed around over-broad references. With the taxonomy stable (three
concrete kinds: APLArray, Function, Operator), those methods belong
on the kinds that use them:

- dop methods moved to `Operator`.
- apply/call/as_power_strategy removed from `APLValue` (Function has
  them; APLArray has its own as_power_strategy).
- `Executable` delegation narrows explicitly via `_as_function` /
  `_as_operator` helpers, producing sharper error messages.
- `PowerByConvergence.test_fn` tightened `APLValue → Function`.
- `MonadicDopCall.op_name` tightened `Applicable → Executable`.

`APLValue` is now just `name_class()`.

## SysVar split + ⎕NC fix (v0.8.39)

**Refactor + bug fix**. Single `SysVar` class was doing double duty
(readable for ⎕IO; callable for ⎕NC). New split:

- `SysVar(Executable)` — readable values only.
- `SysFunc(Function, Executable)` — callable system functions; `execute`
  returns self, `apply_monadic`/`apply_dyadic` dispatch.

Cleanup consequence: `Function.call_monadic` / `call_dyadic` and
`Executable.call_monadic` / `call_dyadic` all deleted — they were
pure delegation that existed only so SysVar's dual identity could
polymorphise.

Bug fix: `⎕NC '⎕IO'` and `⎕NC '⎕NC'` now return ¯1 (Dyalog's "invalid
name" for ⎕-prefix queries). marple previously returned 0 and 3.
Failing tests added first; refactor + `_sys_nc` prefix-check made
them pass.

## Refactoring notes for future work

- `Evaluatable.execute(ctx) -> APLValue` return type is still broad;
  call sites narrow via `_as_function` / `_as_operator`. Could be
  narrowed further by execute-return-type tightening per subclass,
  but the isinstance checks at Executable's boundary are honest.
- N-wise dyadic reduce (`2+/1 2 3 4` → Dyalog `3 5 7`, marple `10`)
  is a known divergence, out of scope for today.
- Error-class precision (RANK ERROR vs DOMAIN ERROR for axis
  out-of-range etc.) may be worth tightening.
