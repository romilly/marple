# Progress Report -- 2026-04-13

## Summary

Major type-system and code-quality refactoring session. Version 0.8.11
to 0.8.20 across 31 commits, 28 files changed (+788, -657). isinstance
count in nodes.py reduced from 35 to 16 (9 are __eq__). Overall
codebase isinstance+hasattr reduced from 177 to 160. 1324 tests
passing, 0 pyright errors.

## Parser stack: Adverb, Conjunction, AssignmentArrow (v0.8.12)

Eliminated raw strings from the parser stack by wrapping them in Node
subclasses using Iverson's terminology. Monadic operators are `Adverb`,
dyadic operators are `Conjunction`. Wrappers stay wrapped downstream
through BoundOperator, DerivedFunc, ReduceDerived, ScanDerived -- only
unwrapped at leaf dispatch points. `StackItem` alias removed -- it's
just `Node` now.

## Narrow Any type annotations (v0.8.13-v0.8.15)

Systematic replacement of `Any` across 10+ files:

- **Callable types**: dispatch dicts and lambda params in nodes.py,
  operator_binding.py, numpy_array.py
- **NDArray types**: backend_functions.py functions narrowed from Any
  to NDArray (new TypeAlias for npt.NDArray[Any]). Removed is_ndarray
  (dead code), _to_python (dead code), dtype_hint parameter (dead code),
  and tests for defensive behaviour against non-ndarray callers
- **Domain types**: formatting.py env parameter, namespace.py entries,
  symbol_table.py values, environment.py locals -- all narrowed to
  concrete types (Environment, APLValue, etc.)
- **Symbol table split**: Single _values dict replaced with typed
  _arrays, _functions, _operators dicts. arrays() returns
  list[tuple[str, APLArray]] without isinstance guard. classify()
  moves values between dicts on reclassification.
- **__wsid__ removed**: Redundant with quad WSID. Eliminated from
  workspace serialisation and live environment.

## Backend cleanup

- **Overflow handling**: Replaced try/except FloatingPointError retry
  pattern with upcast-before-operation approach (Dyalog's rule).
- **Numpy reshape**: Replaced flat offset arithmetic with reshape and
  enumerate. _scan_row_accumulate uses ufunc.accumulate(axis=1).
- **Dtype comparisons**: Replaced string-based dtype checks with
  np.dtype objects and np.issubdtype. data_type_code uses dict lookup.
- **Identity elements**: Fixed ceiling/floor identity from +/-inf to
  +/-float64.max (matches Dyalog).
- **to_bool_array**: Simplified to single np.asarray call.
- **Inlined _apl_chars_to_str** (trivial wrapper around chars_to_str).

## evaluate return type (v0.8.16)

- Tightened Evaluatable.execute from `object` to `APLValue`
- evaluate() asserts APLArray; function-expecting callers use
  execute() directly
- Fixed ExecutionContext protocol conformance
- Eliminated all type:ignore comments from executor.py

## Applicable superclass (v0.8.17)

New `Applicable` ABC between `Node` and `Evaluatable`/`UnappliedFunction`:
- apply_to_monadic/apply_to_dyadic -- apply with evaluated args
- call_monadic/call_dyadic -- apply with unevaluated args
- apply_monadic_dop/apply_dyadic_dop -- operator application
- as_power_strategy -- returns PowerByCount or PowerByConvergence

Removed apply_func_monadic/apply_func_dyadic from Executor and
ExecutionContext protocol entirely. MonadicDfnCall/DyadicDfnCall
collapsed from 12/10 lines to 4/4. MonadicDopCall/DyadicDopCall
no longer import DfnBinding.

## APLValue extraction (v0.8.19-v0.8.20)

- Extracted APLValue and NC_* constants into apl_value.py to break

- PowerStrategy/PowerByCount/PowerByConvergence moved to apl_value.py

- PowerDerived._resolve_strategy collapsed to single line
