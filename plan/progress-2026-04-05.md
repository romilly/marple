# Progress Report 2026-04-05

## Summary

Major refactoring day. Split backend.py into clean modules, migrated from flat data with separate shape to properly shaped numpy arrays, added massive performance improvements to inner/outer product, and fixed dozens of structural operations for shaped data. Version 0.7.0 → 0.7.1.

## What was done

### Module restructuring

- Extracted `get_numpy.py` (numpy/cupy/ulab import chain)
- Extracted `backend_functions.py` (to_array, to_list, is_numeric_array, etc.)
- Moved NumpyArray to `numpy_array.py`, renamed to APLArray (no more ABC)
- Created `formatting.py` (format_num, format_result)
- Deleted `backend.py` (was 884 lines, now gone)
- Deleted `arraymodel.py` (unnecessary re-export layer)
- Updated all imports across 55+ files

### Shaped numpy arrays

- `to_array()` now handles nested lists correctly (creates shaped numpy arrays)
- `reshape` uses `np.concatenate` for cycling, produces shaped results
- `encode` produces shaped 2D results for vector right argument
- `transpose` uses `.T` on shaped data
- `reverse` / `reverse_first` use `np.flip`
- `rotate` / `rotate_first` use `np.roll`
- `catenate` uses `np.concatenate` with axis parameter
- `matrix_inverse` uses `np.linalg.inv`
- `matrix_divide` uses `np.linalg.solve`
- `ravel` uses `.flatten()`
- `from_array` flattens before indexing, reshapes result
- `replicate_first` uses `np.concatenate` for result
- `decompose` reshapes cell data to match cell shape
- `reassemble` concatenates numpy arrays and reshapes
- Indexing node flattens data, uses int() offsets for ulab

### Inner/outer product rewrite

- Generalized inner product to arbitrary rank using index striding
- No more `S()` wrapping — dispatch on glyph using scalar ops
- `np.dot` fast path for `+.×` (390x faster on 100×100 matrices)
- `ufunc.outer` fast path for 13 operations (120-160x faster)
- Pre-allocated numpy result arrays instead of list append

### Scan/reduce rewrite

- `_scan`: numpy `accumulate` for commutative ops, O(n²) loop for non-commutative
- `_reduce`: scalar ops on flattened data, no `S()` wrapping
- `_reduce_first` / `_scan_first`: same treatment
- Rejected dfn operands in scan/reduce (DOMAIN ERROR)

### Dead code removal

- Removed `HAS_BACKEND`, `_backend_name`, `maybe_downcast_scalar`
- Removed `needs_backend` test markers
- Removed dead Python fallback branches from monadic methods
- Deleted `test_numpy_array.py` (implementation tests, all duplicated by behavioural tests)
- Deleted `test_array_factory.py` (isinstance tests that can't fail)

### ulab compatibility

- `np_reshape()` wrapper converts list shapes to tuples
- `.flatten()[0]` instead of `.flat[0]` (ulab flatiter not subscriptable)
- `dtype=float` → no explicit dtype or `np.float` (ulab rejects Python `float`)
- `np.concatenate(tuple(...))` (ulab requires tuple not list)
- Guard `_ACCUMULATE_UFUNCS` and `ufunc.outer` (ulab has no ufunc objects)
- `int()` for array sizes and offsets
- `np.concatenate` instead of `np.tile` (ulab has no tile)

### CLAUDE.md rules added

- Problem handling: never move on without user approval
- Bulk changes: always commit before bulk replacements

## Architecture after refactoring

```
get_numpy.py          — cupy/numpy/ulab import
backend_functions.py  — to_array, to_list, is_numeric_array, np_reshape, etc.
numpy_array.py        — APLArray class (concrete, no ABC) and S()
formatting.py         — format_num, format_result
structural.py         — structural operations (reshape, take, drop, etc.)
operator_binding.py   — reduce, scan, derived function dispatch
nodes.py              — AST nodes, inner/outer product
cells.py              — rank decompose/reassemble
```
