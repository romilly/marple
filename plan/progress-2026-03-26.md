## What was built this session

### Parser rewrite: Iverson's stack-based algorithm

Replaced `_parse_statement()` (190 lines, 15 ad-hoc branches) with a
principled stack-based parser following Iverson's Dictionary of APL:

- **4-position window** on a stack, processing tokens right-to-left
- **9 core cases** from Iverson's algorithm (monadic/dyadic function
  application, adverb/conjunction binding, assignment, parentheses, shift)
- **3 extra cases** for callable noun handling (‚ćļ‚ćļ/‚ćĶ‚ćĶ, dynamically-defined
  functions inside dfn bodies, qualified namespace names)
- **`BoundOperator` AST node** ‚ÄĒ intermediate form for operator binding,
  resolved into final AST nodes when the derived verb is applied
- **Pre-processing** (`_build_items`) converts tokens to classified items:
  dfn bodies, vectors, bracket indices, compound operators (‚ąė.f), i-beam (‚Ć∂'path')
- **Token categories**: END, NOUN, VERB, ADV, CONJ, ASSIGN, LP, RP

Key binding fixes:
- `{‚ćļ+‚ćĶ}/‚ć≥5` now correctly parses as reduce with dfn operand (was: replicate error)
- `{‚ćļ+‚ćĶ}\‚ć≥5` ‚ÄĒ scan with dfn operand
