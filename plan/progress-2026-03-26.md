# MARPLE Progress Report — 27 March 2026

## Summary

Eighth session. Replaced the recursive descent parser with Iverson's
stack-based parsing algorithm from the Dictionary of APL. This gives
correct operator binding precedence — operators now bind tighter than
functions, as required by APL. Also fixed scalar replicate and added
a new demo video.

## What was built this session

### Parser rewrite: Iverson's stack-based algorithm

Replaced `_parse_statement()` (190 lines, 15 ad-hoc branches) with a
principled stack-based parser following Iverson's Dictionary of APL:

- **4-position window** on a stack, processing tokens right-to-left
- **9 core cases** from Iverson's algorithm (monadic/dyadic function
  application, adverb/conjunction binding, assignment, parentheses, shift)
- **3 extra cases** for callable noun handling (⍺⍺/⍵⍵, dynamically-defined
  functions inside dfn bodies, qualified namespace names)
- **`BoundOperator` AST node** — intermediate form for operator binding,
  resolved into final AST nodes when the derived verb is applied
- **Pre-processing** (`_build_items`) converts tokens to classified items:
  dfn bodies, vectors, bracket indices, compound operators (∘.f), i-beam (⌶'path')
- **Token categories**: END, NOUN, VERB, ADV, CONJ, ASSIGN, LP, RP

Key binding fixes:
- `{⍺+⍵}/⍳5` now correctly parses as reduce with dfn operand (was: replicate error)
- `{⍺+⍵}\⍳5` — scan with dfn operand
- `,⍤0 -a` — conjunction binds rank spec before minus can claim it
- `(double) twice 5` — adverb binds before function application

### Interpreter extensions

- `DerivedFunc.function` now accepts `Dfn`, `Var`, `FunctionRef` (not just strings)

- `_newlines_to_diamonds()` converts `\n` to `⋄` in source before parsing

### Scalar replicate fix

### ⎕FMT (Format)

Dyalog-compatible format function with special `(val1;val2;...)` syntax:

- **Format codes**: I (integer), F (fixed-point), E (scientific), A (alpha), G (pattern)
- **Text insertion**: `⊂text⊃` or `<text>` inserts literal text between columns
- **G pattern**: `G⊂99/99/99⊃` fills `9` placeholders with digits (dates, phone numbers)
- **Character matrix result**: vector arguments produce one row per element
- **Format cycling**: format specs repeat across columns
- **Repeated formats**: `5A1` takes 5 chars from one column (not 5 columns)
- **Error validation**: type mismatches, ill-formed specs, decimal overflow
- **Special parsing**: `⎕FMT (` triggers semicolon-separated argument parsing via `FmtArgs` AST node
- 22 tests covering all format codes, character/numeric data, matrices, errors
