## What we're stuck on

### Blocker: ulab has no uint32

Manual test on the Pico reported:

```
File "marple/backend_functions.py", line 26, in str_to_char_array
AttributeError: 'module' object has no attribute 'uint32'
```

Query of ulab dtypes confirmed:

```
['bool', 'float', 'get_printoptions', 'int16', 'int8', 'interp',
 'set_printoptions', 'uint16', 'uint8']
```

The entire char migration chose uint32 as the character representation.
ulab does not expose uint32; its numpy maxes out at 16-bit integer
types. `str_to_char_array` fails on import-time for any test that
touches character data, which is nearly everything via `âŽ•A`, `âŽ•D`,
`âŽ•WSID`, `âŽ•LX` defaults in `environment.py`.

## Proposed next step

**Switch the character representation from uint32 to uint16** throughout
the codebase. uint16 covers 0â€“65535, which includes all of the Basic
Multilingual Plane â€” every APL glyph and every common character.
Astral characters (emoji, historical scripts, U+10000+) would no
longer fit, but that's not realistic for APL source code.

The change is mechanical but wide:

- `grep -rn "uint32"` shows ~60 hits across src/ and tests/.
- `str_to_char_array`, `is_char_array`, `char_fill`,
  `APLArray.__eq__`, `_format_matrix`, `_sys_ucs`, `_sys_dr` etc.
  all have uint32 literals.
- Tests that construct char arrays via `str_to_char_array` need no
  change (they use the helper).
- Tests that check dtype explicitly (`str(data.dtype) == 'uint32'`)
  need to be updated to `'uint16'`.

Suggested sequence:

1. Update `str_to_char_array` to produce uint16.
2. Update `is_char_array` to check `'uint16'`.
3. Update `is_numeric_array` to exclude `'uint16'` instead of `'uint32'`.
4. Update `char_fill` to return `np.uint16(32)`.
5. Update `APLArray.__eq__` (if it mentions uint32 â€” it may not).
6. Run full desktop suite. Expect fallout from tests that check dtype
   strings.
7. Deploy to Pico. Run TestPicoReplicateExpand.
8. Discover whether `np.repeat`, `np.full`, fancy indexing work on ulab.
   If not, fall back to the loop-based replicate/expand (option B3
   from the audit).
