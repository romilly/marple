# MARPLE Progress Report — 25 March 2026

## Summary

Sixth session. Migrated web REPL to aiohttp with WebSocket communication,
added Pico serial bridge for evaluating APL on the Pico from the browser.
Updated documentation.

## What was built this session

### Web REPL migration to aiohttp
- Replaced `http.server.HTTPServer` with `aiohttp.web.Application`
- All 36 existing Playwright tests pass unchanged
- Async request handlers for all HTTP endpoints

### WebSocket communication
- `/ws` endpoint for persistent bidirectional communication
- JSON protocol: `{"type": "eval", "expr": "..."}` and `{"type": "system", "cmd": "..."}`
- Server sends result HTML + workspace updates after each expression
- Frontend switched from `fetch()` POST to WebSocket with auto-reconnect
- Connection status indicator in header bar
- 7 new WebSocket tests using `pytest-aiohttp`

### Pico serial bridge
- `src/marple/web/pico_bridge.py` — PicoConnection class promoted from test fixture
- WebSocket mode switching: `{"type": "mode", "mode": "local|pico"}`
- In Pico mode, expressions sent over USB serial via hex-encoded protocol
- Serial I/O runs in asyncio executor (thread pool) to avoid blocking
- `GET /config` endpoint tells frontend whether Pico is available
- Frontend shows Local/Pico radio toggle when `--pico-port` is specified
- 4 new WebSocket tests (mode switching, mock Pico eval)

### Documentation updates
- Rewrote `docs/reference/system/system-variables.md` — now covers all system
  variables and functions: ⎕IO, ⎕CT, ⎕PP, ⎕RL, ⎕FR, ⎕WSID, ⎕DR, ⎕EA, ⎕UCS,
  ⎕NC, ⎕EX, ⎕SIGNAL, and read-only variables ⎕A, ⎕D, ⎕TS, ⎕EN, ⎕DM, ⎕VER
- Updated system features index page
- Updated README with WebSocket REPL and Pico bridge usage

### Server startup

```bash
# Local only
python -m marple.web.server

# With Pico
python -m marple.web.server --pico-port /dev/ttyACM0
```
