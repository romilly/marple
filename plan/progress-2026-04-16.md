## Bug fix: dyadic `⍕` on rank 2+ arrays (v0.8.71 -> v0.8.72)

`10 2⍕10 1⍴*⍳10` crashed with "sequence item 0: expected str instance,
list found". `dyadic_format` flattened the input and joined all
formatted strings, but `strs.tolist()` on a 2D numpy string array
returns nested lists. Fixed to preserve all leading dimensions and
transform only the last axis (`last_dim * width`), matching Dyalog
for any rank. Verified against Dyalog for rank 2 and rank 3.

## Bug fix: `∧` and `∨` are LCM/GCD (v0.8.74)

The implementations of `∧` (logical_and) and `∨` (logical_or) treated
them as boolean AND/OR everywhere. In Dyalog, they are LCM and GCD
for all numeric arguments (matching AND/OR only for boolean 0/1 inputs).
`3∧4` returned 1 instead of 12; `6∨4` returned 1 instead of 2.

Fixed in three places: `logical_and`, `logical_or`, and
`_INNER_SCALAR_OPS` (used by inner/outer products). Integer fast path
uses `np.gcd`; float path uses Euclidean algorithm with tolerance.
Verified against Dyalog for integers, floats, and outer products.
