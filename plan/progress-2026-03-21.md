# MARPLE Progress Report — 21 March 2026

## Summary

In a single session, MARPLE went from an empty project skeleton to a working
first-generation APL interpreter with an interactive REPL. All four planned
implementation phases are complete.

## What was built

### Core interpreter (1,905 lines across 11 modules)

- **Array model** — `APLArray(shape, data)` inspired by Rodrigo Girão Serrão's
  RGSPL, supporting scalars, vectors, and matrices
- **Tokenizer** — handles numbers, high minus (¯), 40+ APL glyphs, strings,
  system variables (⎕IO), bracket indexing, dfn syntax
- **Parser** — right-to-left recursive descent with long right scope and short
  left scope, matching APL's evaluation rules
- **Interpreter** — tree-walking evaluator with lexically scoped dfn closures

### Language features

| Phase | Features |
|-------|----------|
| **1 — Calculator** | Scalars, vectors, `+ - × ÷ ⌈ ⌊`, right-to-left eval, parens, assignment (←), comments (⍝) |
| **2 — Useful subset** | Statement separator (⋄), extended scalars (`* ⍟ \| ○ < ≤ = ≥ > ≠ ∧ ∨ ~`), structural functions (`⍴ ⍳ , ↑ ↓ ⌽`), reduce (`/`), scan (`\`) |
| **3 — Dfns** | `{⍵}` syntax, `⍺` left arg, guards (`cond:expr`), `∇` recursion, `⍺←default`, named functions via assignment |
| **4 — Full first-gen** | Matrices, bracket indexing (`M[r;c]`), transpose (`⍉`), grade (`⍋ ⍒`), encode/decode (`⊤ ⊥`), inner product (`f.g`), outer product (`∘.f`), execute/format (`⍎ ⍕`), replicate/expand (`/ \`), matrix inverse/divide (`⌹`), circular functions (`○`), `⎕IO` |

### REPL

- Interactive terminal with live backtick→glyph translation (`` `r `` → `⍴`)
- Silent assignment (APL convention)
- Right-aligned column formatting for numeric matrices
- Correct character array display (no spurious spaces)
- System commands: `)off`, `)clear`, `)save`, `)load`, `)fns`, `)vars`
- Workspace save/load (variables, dfns, system variables persisted as APL text)
- Version display at startup

### Test suite

- **229 tests**, all passing
- **0 pyright errors** (strict mode)
- Tests cover: array model, tokenizer, parser, all function categories,
  operators, dfns, integration, display formatting, workspace persistence

## Design decisions

- **Pure Python** — Rust was dropped from the original spec; no external
  dependencies beyond the standard library (test deps: pytest, pyright, PyHamcrest)
- **APL arrays over JSON** — `APLArray(shape, data)` replaces the originally
  planned JSON/base64 encoding, following RGSPL's design
- **Rewrite inspired-by** — RGSPL was used as a design reference (MIT licensed)
  but all code was written from scratch via TDD
- **No nested arrays** — flat arrays only, as specified for first-generation APL
- **Dfns over tradfns** — modern direct definition syntax rather than the ∇ editor
