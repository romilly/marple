# Progress Report 2026-04-01

## Summary

Major session focused on Console/REPL unification, Pico connection reliability, and bug fixes across the parser, PRIDE, workspace system, and deploy pipeline. Version 0.5.36 → 0.5.43.

## What was done

### REPL multi-line input (brace balancing)

- `run_repl` now accumulates lines until braces are balanced before calling `interp.execute()`
- Uses the existing `is_balanced()` function from the parser
- Extracted shared `process_line()` helper used by both `run_repl` and `run_script`, eliminating duplicated accumulate/execute logic

### Parser: unbalanced braces hang

- Discovered the parser hangs on unbalanced braces (e.g. `double←{`)
- Added a test that documents the hang (thread-based timeout)
- Not yet fixed — the REPL now guards against it via `is_balanced()`

### Parser: silent discard of unreduced stack items

- **Critical bug fixed**: the parser silently discarded unreduced items from its stack, masking errors
- `1 0 1 0 0 nosuchfn ⍳5` returned `1 0 1 0 0` instead of raising an error
- Now raises `SYNTAX ERROR: Expression could not be fully parsed`
- Also fixed: `{⍺⍺+⍵} 5` (operator without left operand) was silently returning S(0), now correctly errors

### Workspace save: empty ⎕LX

- `save_workspace` wrote `⎕LX←` (no value) for empty latent expressions
- This caused the parser to error/hang when the workspace was loaded
- Fixed: skip saving when formatted value is empty

### PicoConnection: probe-based readiness

- Replaced the `\x02` ready signal with a probe-and-wait approach
- PicoConnection sends an empty line and waits for the sentinel response
- Handles all Pico states: not connected, booting, already running
- Removed `READY` constant from `pico_eval.py`
- 3 new unit tests for probe logic

### PRIDE: system commands in pico mode

- **Bug fixed**: system commands (`)load`, `)lib`, etc.) always ran on the local interpreter, even in pico mode
- `_handle_system` now checks `self.mode == "pico"` and sends commands to the Pico

### )DROP command

- New system command to delete a workspace
- Added `delete_dir` method to FileSystem port and all adapters (OsFileSystem, FakeFileSystem)
- Works on both CPython and MicroPython (recursive file delete + rmdir)
- 3 unit tests + Pico E2E test (save/load/drop cycle)

### Deploy script improvements

- Workspaces now deployed to Pico (life, partfns)
- Workspace deployment happens before reset (was failing due to Pico reboot)
- Dotfiles (`.ws` marker) now included in workspace deployment
- Removed unnecessary temp directory copy (was leftover from removed `__future__` stripping)
- Deploys directly from `src/marple/`

### Dependency pinning

- All dependencies pinned to exact versions (`==`) in pyproject.toml, requirements.txt, requirements-test.txt
- Added rule to CLAUDE.md: always pin dependencies (supply-chain protection)
- Updated Python classifiers: removed 3.8/3.9, added 3.13

### )lib output

- Empty workspace list now returns empty string instead of "(none)"
