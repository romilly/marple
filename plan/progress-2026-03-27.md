# MARPLE Progress Report — 27 March 2026

## Summary

Ninth session. Parser rewrite, ⎕FMT, three new system functions, Presto
migration, and a major refactoring pass replacing isinstance chains with
dispatch dicts throughout the codebase.

## What was built this session

### Parser rewrite: Iverson's stack-based algorithm

Replaced the 15-branch recursive descent `_parse_statement()` with a
principled stack-based parser following Iverson's Dictionary of APL:

- 9 core pattern-matching cases + 3 callable-noun cases
- Correct operator binding precedence: operators bind tighter than functions
- `{⍺+⍵}/⍳5` now works (dfn as reduce operand — previously crashed)
- `BoundOperator` intermediate AST node for operator binding
- Pre-processing handles `∘.f`, `⌶'path'`, bracket indexing, dfn bodies
- `_newlines_to_diamonds()` conversion in `interpret()`

### ⎕FMT (Format)

Dyalog-compatible format function with special `(val1;val2;...)` syntax:

- Format codes: I (integer), F (fixed-point), E (scientific), A (alpha),
  G (pattern with `⊂99/99⊃` digit placeholders)
- Text insertion: `⊂text⊃` and `<text>` delimiters
- Character matrix result; vector arguments produce rows
- Repeated formats: `5A1` takes 5 chars from one column (not 5 columns)
- Format cycling across columns
- `FmtArgs` AST node with semicolon-separated parsing
- Error validation: type mismatches, decimal overflow, bad specs
- 22 tests

### New system functions

- **⎕DL** — delay n seconds, returns elapsed time
- **⎕NL** — name list by class (2=var, 3=fn, 4=op) as character matrix
- **⎕CSV** — import CSV file, create workspace variables from header columns
  (numeric vectors or character matrices), returns row count

### Presto migration

Migrated from Pimoroni Explorer (320x240) to Pimoroni Presto (480x480):
- `PrestoDisplay` class in `scripts/presto_display.py`
- Tested and working via PRIDE web bridge

### Dispatch-dict refactoring

Replaced isinstance/elif chains with dispatch dicts across the codebase:

- **interpreter.py**: `_EVAL_DISPATCH` (26 AST handlers),
  `_MONADIC_SYS`/`_DYADIC_SYS`, dfn-call dispatch, runtime value dispatch.
  isinstance count reduced from 54 to 22.
- **repl.py**: `_SYSTEM_COMMANDS` dict (8 commands)
- **web/server.py**: `_SYS_COMMANDS` dict (4 commands)

### Terminology fix

Renamed `_DfnClosure` → `_DfnBinding` throughout. Purged all references to
"closures" and "lexical scope" from docs — MARPLE dfns use dynamic lookup.

### Bug fixes

- Dyadic dop derived verb: `⍺` now passed through `MonadicDopCall`
- Case-insensitive quad-names: `⎕nl` works same as `⎕NL`
- `AlphaAlpha`/`OmegaOmega` in `_make_dyadic` (was missing)
