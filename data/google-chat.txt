Project Summary: MARPLE (Microprocessor-friendly APL)The Vision
MARPLE is a modern, standards-compliant APL interpreter designed to run everywhere—from the high-performance Linux desktop to the Raspberry Pi Pico 2 (RP2350). Created by a veteran of the APL community who worked alongside pioneers like Ken Iverson and Ian Sharp, MARPLE is the realization of a lifelong vision for the "perfect" APL.
Key Innovations

* Architecture: Built using a Hexagonal Architecture (Ports and Adapters), allowing the core APL logic to remain identical while swapping out the computational "engine."
* Backends:
* Pure Python: For portability.
   * NumPy: For high-performance array operations on desktop.
   * ulab-numpy: A C-based NumPy subset for microcontrollers (MicroPython).
   * JAX (In-Progress): To enable GPU acceleration and JIT compilation, targeting Artificial Neural Networks (ANNs).
* AI-Directed Development: The entire codebase, documentation, and test suite (~1,000 automated tests) were generated via Claude Code under the user’s expert direction and rigorous refactoring.
* Hardware Integration: Designed for "bare metal" use on the Pico 2 using a custom I2C-based keyboard (Dyalog layout) with a clever Shift-hold toggle for APL symbols.

The Pedigree
The creator founded the UK APL User Group and has a deep history with IBM, Sharp APL, and Dyalog. MARPLE bridges the gap between the 1960s "Universal Notation" and modern machine learning/embedded systems.
Features

* Standard Compliance: ISO/ANSI APL compatible.
* Modern Extensions: Full support for dfns (functional APL) and dops (defined operators).
* Persistence: Supports )SAVE and )LOAD even on microcontrollers.
* Efficiency: Strategic use of vectorised libraries ensures performance comparable to industry-standard interpreters.
