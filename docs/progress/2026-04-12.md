## Refactoring (v0.8.7–v0.8.11)

### FunctionRef at source (v0.8.7)
Wrap all function glyphs in FunctionRef at `_item_function` instead
of scattered wrapping points. Removed isinstance(str) branches from
executor and operator_binding. Removed dead ReduceOp/ScanOp classes
and resolve_dyadic method.

### APLValue/UnappliedFunction hierarchy (v0.8.8)
APLValue ABC in symbol_table.py with abstract name_class(). APLArray
and UnappliedFunction both inherit from it. _name_class collapsed
from 9-type isinstance chain to `value.name_class()`.

### Polymorphic apply_monadic/apply_dyadic (v0.8.8–v0.8.9)
Each UnappliedFunction subclass (RankDerived, PowerDerived,
CommuteDerived, BesideDerived, AtopDerived, ForkDerived, DfnBinding,
FunctionRef, IBeamDerived) now implements apply_monadic/apply_dyadic.
MonadicDfnCall and DyadicDfnCall collapsed from ~46 lines of
isinstance dispatch to 4 lines each.

### ReduceDerived/ScanDerived (v0.8.9)
New UnappliedFunction subclasses for reduce (f/ f⌿) and scan (f\ f⍀).
Parser resolves all BoundOperator operands via _resolve_operand helper.
BoundOperator completely eliminated from executor.

### Move apply logic into subclasses (v0.8.10)
Commute, Beside, Atop, Fork, Rank, Power apply logic moved from
executor methods into the UnappliedFunction subclasses. Executor
lost ~270 lines of operator-specific methods. Literal wrapper node
bridges the evaluated/unevaluated interface gap.

### Node/Evaluatable type hierarchy (v0.8.11)
Node is now the base for all parser stack items. Evaluatable (Node
subclass with abstract execute) is the base for executable AST nodes.
BoundOperator, Marker, FmtArgs, Guard, AlphaDefault all Node subclasses.
All node fields, parser methods, and executor signatures tightened
from `object` to proper types. Zero pyright errors achieved.
