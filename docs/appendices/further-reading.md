# Further Reading

## Foundational texts

- Ken Iverson. *A Dictionary of APL*. APL Quote Quad 18(1), September 1987. [Online](https://www.jsoftware.com/papers/APLDictionary.htm). The theoretical foundation for MARPLE's design.
- Ken Iverson. *Rationalized APL*. IPSA, 1983. Introduces rank and the leading-axis approach.
- Robert Bernecky. "An Introduction to Function Rank." APL88 conference proceedings, 1988. The best introduction to why rank matters.
- Ken Iverson. *Notation as a Tool of Thought*. Communications of the ACM, 1980. [Online](https://www.jsoftware.com/papers/tot.htm). The Turing Award lecture that set out the case for array notation.

## The rank operator

- Arthur Whitney. *Compact Hash Tables* and early rank work at I.P. Sharp Associates, 1982.
- Roger Hui and Ken Iverson. *J Introduction and Dictionary*. [Online](https://www.jsoftware.com/help/dictionary/intro.htm). Rank is central to J's design.
- Rodrigo Girao Serrao. RGSPL blog posts on implementing rank in Python.

## Learning APL

- APL Wiki: [https://aplwiki.com](https://aplwiki.com) -- comprehensive reference for all APL dialects
- Stefan Kruger. *Learning APL*: [https://xpqz.github.io/learnapl/](https://xpqz.github.io/learnapl/) -- modern tutorial using Dyalog
- Dyalog APL course: [https://course.dyalog.com](https://course.dyalog.com) -- interactive lessons
- APL Orchard chat: [https://apl.chat](https://apl.chat) -- community chat room

## Related projects

- RGSPL: [https://github.com/rodrigogiraoserrao/RGSPL](https://github.com/rodrigogiraoserrao/RGSPL) -- another Python APL, a design reference for MARPLE
- J Software: [https://www.jsoftware.com](https://www.jsoftware.com) -- the language most closely related to MARPLE's design philosophy
- Dyalog APL: [https://www.dyalog.com](https://www.dyalog.com) -- the most widely used commercial APL
- GNU APL: [https://www.gnu.org/software/apl/](https://www.gnu.org/software/apl/) -- free Software Foundation APL implementation

## Standards

- ISO 8485:1989 -- the flat APL standard that MARPLE's core follows

## NumPy and array programming

- NumPy documentation: [https://numpy.org/doc/](https://numpy.org/doc/) -- the backend MARPLE uses for acceleration
- ulab documentation: [https://micropython-ulab.readthedocs.io/](https://micropython-ulab.readthedocs.io/) -- the MicroPython/CircuitPython array library
